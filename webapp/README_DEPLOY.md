# 🌐 Размещение веб-приложения

Этот файл содержит инструкции по размещению веб-приложения для Telegram мини-приложения.

## ⚠️ Важно

Telegram мини-приложения требуют **HTTPS URL**. HTTP не будет работать!

## 🚀 Вариант 1: Netlify (Рекомендуется - самый простой)

1. Перейдите на https://app.netlify.com
2. Зарегистрируйтесь (можно через GitHub - бесплатно)
3. На главной странице найдите секцию "Sites"
4. Перетащите всю папку `webapp` на страницу Netlify
5. Через 10-30 секунд получите URL вида: `https://random-name-123.netlify.app`
6. Скопируйте этот URL и используйте в `bot.py`

**Преимущества:**
- ✅ Бесплатно
- ✅ Автоматический HTTPS
- ✅ Работает сразу
- ✅ Не требует настройки

## 🚀 Вариант 2: Vercel

1. Установите Vercel CLI: `npm install -g vercel`
2. Откройте терминал в папке `webapp`
3. Выполните: `vercel`
4. Следуйте инструкциям
5. Для продакшена: `vercel --prod`

## 🚀 Вариант 3: GitHub Pages

1. Создайте новый репозиторий на GitHub
2. Загрузите все файлы из папки `webapp` в репозиторий
3. Перейдите в Settings → Pages
4. Source: выберите `main` branch и папку `/ (root)`
5. Сохраните
6. Через минуту получите URL: `https://yourusername.github.io/repository-name/`

## 🧪 Локальное тестирование (только для разработки)

Для тестирования в браузере (не для Telegram):
```bash
python server.py
```

Откройте http://localhost:8000

**Для Telegram нужен HTTPS!** Используйте ngrok:
1. Установите ngrok: https://ngrok.com/download
2. Запустите: `python server.py` (в папке webapp)
3. В другом терминале: `ngrok http 8000`
4. Скопируйте HTTPS URL из ngrok

## 📝 После размещения

1. Скопируйте HTTPS URL вашего приложения
2. Откройте `bot.py` в корне проекта
3. Найдите строку: `WEB_APP_URL = "https://your-domain.com/index.html"`
4. Замените на ваш реальный URL
5. Сохраните файл

## ✅ Проверка

Откройте ваш URL в браузере - должно открыться приложение с кейсами.
Если видите интерфейс - всё готово!
