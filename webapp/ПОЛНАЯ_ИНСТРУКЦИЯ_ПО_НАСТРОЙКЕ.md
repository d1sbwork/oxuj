# 📋 ПОЛНАЯ ИНСТРУКЦИЯ ПО НАСТРОЙКЕ АВТОМАТИЧЕСКОГО ДЕПЛОЯ

## 🎯 Цель

Настроить систему, которая **автоматически** деплоит любые изменения в Telegram мини-приложение **без вашего участия**.

---

## ✅ ПРОВЕРКА ПЕРЕД НАЧАЛОМ

### Шаг 0: Проверка установленного ПО

**Проверьте, что установлено:**

1. **Python** (обязательно)
   - Откройте командную строку (Win+R → `cmd`)
   - Введите: `python --version`
   - Должно показать версию (например: Python 3.11.0)
   - ❌ Если ошибка → Скачайте: https://www.python.org/downloads/
   - ✅ При установке отметьте "Add Python to PATH"

2. **Git** (обязательно)
   - В командной строке: `git --version`
   - Должно показать версию (например: git version 2.40.0)
   - ❌ Если ошибка → Скачайте: https://git-scm.com/download/win
   - ✅ При установке используйте настройки по умолчанию

3. **Аккаунт GitHub** (обязательно)
   - Зарегистрируйтесь на: https://github.com/
   - Или войдите, если уже есть аккаунт

4. **Аккаунт Netlify** (обязательно)
   - Зарегистрируйтесь на: https://app.netlify.com/
   - Можно войти через GitHub

## 📝 ПОШАГОВАЯ НАСТРОЙКА

### ШАГ 1: Настройка Git и GitHub (5 минут)

#### 1.1. Запустите скрипт настройки

1. Откройте папку: `C:\Users\Alex\Desktop\Case\webapp`
2. Запустите файл: **`БЫСТРАЯ_НАСТРОЙКА_GITHUB.bat`** (двойной клик)

#### 1.2. Следуйте инструкциям на экране

Скрипт проверит и настроит:

**Если Git не установлен:**
- Скрипт покажет ошибку
- Скачайте Git: https://git-scm.com/download/win
- Установите и запустите скрипт снова

**Если Git не настроен:**
- Введите ваше имя (например: "Alex")
- Введите ваш email (например: "alex@example.com")

**Если репозиторий не создан:**
- Скрипт автоматически создаст его

**Если GitHub не подключен:**
- Скрипт попросит URL репозитория
- Создайте репозиторий на GitHub:
  1. Зайдите на: https://github.com/new
  2. Введите название (например: "telegram-miniapp")
  3. Выберите "Public" или "Private"
  4. НЕ отмечайте "Add a README file"
  5. Нажмите "Create repository"
  6. Скопируйте URL (например: `https://github.com/ваш-логин/telegram-miniapp.git`)
  7. Вставьте в скрипт

#### 1.3. Проверка ШАГ 1

После завершения скрипта проверьте:

```bash
# Откройте командную строку в папке проекта
cd C:\Users\Alex\Desktop\Case

# Проверьте Git
git --version
# Должно показать версию

# Проверьте репозиторий
git status
# Должно показать статус файлов

# Проверьте GitHub
git remote -v
# Должно показать URL вашего репозитория
```

✅ **Если всё работает** → Переходите к ШАГ 2

❌ **Если ошибки:**
- Проверьте, что Git установлен
- Проверьте, что репозиторий создан на GitHub
- Запустите скрипт снова

### ШАГ 2: Подключение Netlify к GitHub (5 минут)

#### 2.1. Откройте Netlify

1. Откройте браузер
2. Зайдите на: **https://app.netlify.com/**
3. Войдите в аккаунт (или создайте новый)
   - Можно войти через GitHub (рекомендуется)

#### 2.2. Импортируйте проект

1. Нажмите кнопку: **"Add new site"**
2. Выберите: **"Import an existing project"**
3. Выберите: **"GitHub"**
   - Если первый раз → авторизуйтесь через GitHub
   - Разрешите доступ к репозиториям

#### 2.3. Выберите репозиторий

1. В списке найдите ваш репозиторий
2. Нажмите на него

#### 2.4. Настройте деплой

**ВАЖНО! Правильные настройки:**

```
Base directory:    webapp
Build command:     (оставьте ПУСТЫМ)
Publish directory:  .
```

**Как настроить:**
1. Нажмите "Show advanced"
2. В поле "Base directory" введите: `webapp`
3. Поле "Build command" оставьте пустым
4. В поле "Publish directory" введите: `.` (точка)

#### 2.5. Запустите деплой

1. Нажмите кнопку: **"Deploy site"**
2. Дождитесь завершения деплоя (1-2 минуты)
3. Вы увидите URL вашего сайта (например: `https://your-site.netlify.app`)

#### 2.6. Проверка ШАГ 2

✅ **Проверьте:**
- Сайт открывается по URL от Netlify
- Видны все файлы (index.html, style.css, app.js)
- Нет ошибок в логах деплоя

❌ **Если ошибки:**
- Проверьте настройки Base directory и Publish directory
- Убедитесь, что репозиторий содержит папку `webapp`
- Проверьте логи деплоя в Netlify

---

### ШАГ 3: Настройка автоматического деплоя (2 минуты)

#### 3.1. Выберите способ запуска

**Вариант А: Запуск вручную** (каждый раз при включении ПК)
- Запускайте файл `ЗАПУСТИТЬ_АВТОДЕПЛОЙ_В_ФОНЕ.bat` каждый раз

**Вариант Б: Автозапуск** (РЕКОМЕНДУЕТСЯ - запускается сам)

1. Нажмите `Win + R`
2. Введите: `shell:startup`
3. Нажмите Enter
4. Откроется папка автозагрузки
5. Скопируйте файл `АВТОЗАПУСК_ПРИ_СТАРТЕ.bat` в эту папку
   - Или создайте ярлык на этот файл

#### 3.2. Запустите автоматический деплой

1. Откройте папку: `C:\Users\Alex\Desktop\Case\webapp`
2. Запустите файл: **`ЗАПУСТИТЬ_АВТОДЕПЛОЙ_В_ФОНЕ.bat`**
3. Скрипт запустится в фоне (окно закроется)

#### 3.3. Проверка ШАГ 3

✅ **Проверьте, что скрипт работает:**

1. Откройте файл: `C:\Users\Alex\Desktop\Case\webapp\auto_deploy.log`
2. Должна быть запись: `🚀 Автоматический деплой запущен`

3. Или проверьте процессы:
   - Откройте Диспетчер задач (Ctrl+Shift+Esc)
   - Найдите процесс `pythonw.exe`
   - Если есть → скрипт работает

❌ **Если не работает:**
- Проверьте, что Python установлен
- Проверьте, что файл `auto_deploy_background.py` существует
- Запустите скрипт снова

## 🔧 РЕШЕНИЕ ПРОБЛЕМ

### Проблема: Git не найден

**Решение:**
1. Скачайте Git: https://git-scm.com/download/win
2. Установите с настройками по умолчанию
3. Перезапустите командную строку
4. Запустите скрипт снова

### Проблема: Python не найден

**Решение:**
1. Скачайте Python: https://www.python.org/downloads/
2. При установке отметьте "Add Python to PATH"
3. Перезапустите командную строку
4. Запустите скрипт снова

### Проблема: GitHub не подключается

**Решение:**
1. Проверьте, что репозиторий создан на GitHub
2. Проверьте URL репозитория (должен быть правильным)
3. Попробуйте использовать SSH вместо HTTPS:
   ```bash
   git remote set-url origin git@github.com:ваш-логин/репозиторий.git
   ```

### Проблема: Netlify не деплоит

**Решение:**
1. Проверьте настройки в Netlify:
   - Base directory: `webapp`
   - Publish directory: `.`
2. Проверьте, что репозиторий подключен
3. Проверьте логи деплоя в Netlify

### Проблема: Изменения не появляются в Telegram

**Решение:**
1. Подождите 2-3 минуты (Netlify нужно время)
2. Обновите мини-приложение в Telegram (закройте и откройте)
3. Проверьте URL в BotFather (должен быть правильным)
4. Проверьте логи Netlify

### Проблема: Скрипт не запускается

**Решение:**
1. Проверьте, что Python установлен
2. Проверьте, что файл `auto_deploy_background.py` существует
3. Попробуйте запустить вручную:
   ```bash
   cd C:\Users\Alex\Desktop\Case\webapp
   python auto_deploy_background.py --window
   ```
